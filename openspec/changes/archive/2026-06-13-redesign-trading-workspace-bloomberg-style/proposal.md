## Why

当前看盘页虽然已经接上 bootstrap 与实时图表链路，但首屏把控制、诊断、图表和状态信息同时铺开，阅读路径更像工业控制面板而不是交易型 WebApp。用户的核心任务是先看图、再做低频切换，而不是在一个视口里同时扫描所有面板；因此需要尽快把现有页面重构为图表优先、暗冷现代、支持渐进披露的信息架构。

## What Changes

- 新增一个面向交易工作区体验的 UI capability，定义 Bloomberg 风格启发下的暗冷视觉语言、图表优先布局和渐进披露规则。
- 重构看盘页首屏结构为“顶部命令条 + 主图工作区 + 右侧摘要栏”，把低频设置与诊断信息下沉到第二屏或折叠区。
- 统一前端设计令牌，包括背景、面板、边框、强调色、状态色、字体层级和间距体系，避免当前浅色卡片式后台观感。
- 调整图表容器与指标摘要面板的展示职责，让图表成为绝对视觉中心，同时保留 forming/closed bar 与 provisional/finalized 指标的可见状态提示。
- 定义桌面端与移动端响应式策略，确保页面允许滚动浏览，而不是试图在单一视口内塞满所有信息。
- 保持既有 bootstrap、WebSocket 和指标计算链路不变，本次变更聚焦展示层、交互层和可验证的页面结构约束。

## Capabilities

### New Capabilities
- `trading-workspace-ui`: 定义交易工作区的页面结构、视觉主题、信息层级、响应式行为和渐进披露规则。

## Impact

- 主要影响 `tradingassistant/frontend/app.py`、`tradingassistant/frontend/charting.py` 与相关前端状态组织方式。
- 可能新增前端主题令牌或布局辅助模块，用于复用暗色视觉变量和页面分区样式。
- 不修改后端 API 协议与实时数据语义，但会重新定义这些状态在 UI 中的呈现位置和优先级。
- 需要补充前端验收标准，包括首屏层级、滚动行为、暗色主题一致性、桌面/移动响应式和图表可读性验证。
